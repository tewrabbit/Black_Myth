## 🕹️ 必须实现的核心基础功能 (高层概述)

本项目必须实现一个可运行、不崩溃的版本，具备以下基本功能：

1.  **主角动作系统：** 悟空基础动作（行走、跳跃、闪避）及至少 **3 段基础连招**。
2.  **核心战斗机制：** 玩家与敌人生命值、受击硬直反馈、精确的碰撞和攻击范围判定 (Hitbox)。
3.  **敌人与 Boss：** 至少 **3 种普通怪物** 及 **1 个具备阶段变化（怒气/二阶段）的 Boss**。
4.  **场景与界面：** 支持 3D 场景移动和场景切换；实现玩家/Boss 血条、菜单和死亡重开功能。

## ⚙️ 通用技术与规范要求

本项目强调代码质量和工程规范，必须满足以下条件：

* **版本控制：** 必须使用 Git，并公开发布至指定托管网站，保留清晰的个人提交记录。
* **C++ 特性：** 必须合理使用**至少 3 条**指定 C++ 功能（如 STL 容器、类与多态、C++11/14/17 功能等）。
* **代码规范：** 代码格式统一，命名符合 **Google C++ Style**。
* **代码查重：** 允许重复度 $\le 30\%$。
* **知识产权：** 不得抄袭任何代码，核心逻辑需**自行消化重新编写**，并在文档中注明来源。
